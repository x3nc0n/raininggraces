## Work Summary

Fixed two critical security vulnerabilities in the photo app post-deploy smoke tests:

1. **SWA /admin Route Protection**
   - Issue: Bare `/admin` path not protected (fell through to navigationFallback)
   - Fix: Added exact `/admin` route with `allowedRoles: ["authenticated"]` in `staticwebapp.config.json`
   - Pattern: For every wildcard-protected section (`/admin/*`), pair it with an exact root route
   - Commit: Included in security fixes commit

2. **Azure Functions Rate Limiting Persistence**
   - Issue: In-memory Map rate limit counter not persisting across Function instance cold starts
   - Fix: Migrated to Azure Table Storage (`ratelimits` table, same `straininggraces` account)
   - Implementation: `api/helpers/storage.js` exports `isRateLimited`, `registerFailedAttempt`, `resetRateLimit`
   - Design: PartitionKey=`ratelimit`, RowKey=sanitized-IP, 60-second window, 5-attempt limit
   - Auto-setup: `ensureStorageReady()` creates table on first cold start
   - Commit: Included in security fixes commit
